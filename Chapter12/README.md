## 12 확장성과 아키텍처 패턴

### 12-1-1 Node.js 애플리케이션 확장

논블로킹 i/o 패러다임 덕분이 동시 요청을 처리하는 리소스 사용 최적화  
적당한 수의 요청(일반적으로 초당 수백개)를 처리할 때 훌륭.  
i/o 바인딩 작업에 특히 능함.(파일 시스템 및 네트워크 읽기쓰기)  
cpu바운드(데이터가공)이 아님.  

결국 프로세스를 확대해야할 필요가 있다.
